FEATURES

Functional API for the best content editing tool

Notion is now the best way to run a blog, host a team wiki, write documentation, and much more.

  • Great code support

    We provide libraries to help you implement NotionAPI into your development ecosystem. We also hope to support no-code platforms via NotionAPI Server.

  • Supports most block types

    Most block types, including columns, are supported by NotionAPI, and we are constantly improving and adding support for more blocks.

  • Affordable pricing

    Use NotionAPI Core for free. NotionAPI Server can be hosted on your platform of choice. Also, please support us to continue development!

  • Open-source

    Every package from NotionAPI is completely open source. This means that we accept contributions from other developers, and anyone can introspect how NotionAPI works.

CAPABILITIES

Powerful querying that fits your requirements

With NotionAPI Core and/or Server, you can use Notion as a complete backend solution to get exactly the content you need.

  • Fetch exactly what you need

    Using the NotionAPI Server, you can harness the powers of GraphQL querying. Retrieve everything, or only what you need.

  • Different queries for different blocks

    Notion has a diverse set of different blocks, each with their own schema. This is represented in the data returned by NotionAPI Core and Server.

  • Retrieve files from a collection

    Not only can you now use Notion for text and simple blocks, but you can use it as a full backend file hosting platform.

  • Functional filtering and sorting

    NotionAPI allows for filtering and sorting that is available within the Notion front-end so your data is ordered exactly how you expect.

    • query LatestPosts($limit: Int) { fetchCollection( collectionId: "audhfaiusdyfiuashfdj", collectionViewId: "nmbjhgyut65rerdfxcvbn", filters: [{ property: "Status", operator: "enum_is", value: "Published" }], sort: { property: "Publish date", direction: "descending" }, limit: $limit ) { schema pages { id title properties metadata { format { cover } } } } }
